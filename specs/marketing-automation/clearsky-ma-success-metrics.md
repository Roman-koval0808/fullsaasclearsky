# **30-Day Metrics — System Health and Early Signal**

At 30 days the ClearSky team confirms everything is running correctly and the system is accumulating signal. The client is told: everything is set up and running. Here is what the first month looks like.

## **Internal health checks — ClearSky team verifies all of these**

- Pixel firing on all tracked pages with zero console errors

- All four intent bucket custom events appearing in the event stream

- Engagement threshold filtering correctly — pixel fires only on qualifying sessions

- Four custom audiences populated in Google Ads and Meta with at least one session each

- Anonymous ID persistence confirmed across return sessions

- Communication hub receiving website activity events and writing to prospect profiles

- OAuth connections to Google and Meta confirmed active

## **Early outcome signals — directional only at 30 days**

| **Signal** | **Target** | **What it tells us** |
| --- | --- | --- |
| Intent bucket assignment rate | 70%+ of qualifying sessions receive a bucket | Rules engine is classifying visitors correctly |
| Emergency bucket volume | Meaningful volume if contractor serves emergency work | Traffic quality and intent rules are aligned |
| Return visitor rate | 8–12% of sessions are return visits | On-site recognition is working |
| Forwarded link click-through | At least 1 confirmed from hub outbound | Hub integration and token mechanism are functioning end to end |

## **What the client hears at 30 days**

Everything is running. Your system is watching every visitor and building your audience pools. In the next 30 days those audiences will be large enough to start running targeted ads. Here is what the first month of data looks like.

| *Below-target response: any internal health check failure is a ClearSky team issue to resolve immediately — not communicated to the client as a problem. The client should never know the system had an issue that the team fixed. They should only know it is working.* |

# **60-Day Metrics — First Real Performance Conversation**

By 60 days the system has accumulated approximately 15,000 sessions across the network pool. Basic pattern validation is beginning. The 60-day review is the first time the ClearSky rep talks to the client about actual performance numbers.

## **Internal checks**

- Bucket distribution plausible for a plumbing contractor: Emergency + Active Project 45–60% combined, Comparison 20–30%, Research 15–25%

- Pixel event volume growing week over week

- No bucket assignment errors in the event stream

- OAuth connections still active — token refresh has not silently failed

- ContentRadar publishing on schedule — no missed windows

## **Performance targets**

| **Metric** | **Target** | **Source** | **If below target** |
| --- | --- | --- | --- |
| Emergency audience size | 150–300 members | Google Ads / Meta | Review engagement threshold config — resolve internally before client review |
| Active Project audience size | 150–300 members | Google Ads / Meta | Same as above |
| Return visitor conversion lift | 15–25% above first-visit baseline | ClearSky analytics | Run structured internal review — see protocol below |
| Named hub profiles with full session history | At least 3 | Communication hub | Confirm merge logic triggering at conversion |
| ContentRadar content published | At least 2 pieces live and indexed | ClearSky content log | Escalate to content team immediately |

**Below-target internal protocol at 60 days**

The rep runs a structured internal review before the client call: bucket distribution check, pixel volume trend, OAuth status, intent rules review, ContentRadar publishing log. Issues are resolved before the client call. The client call is a performance conversation, not a troubleshooting session.

## **What the client hears at 60 days**

Your audiences are large enough to run retargeting now. Visitors who come back to your site are converting at a higher rate than first-time visitors — that is the system working. Your content is live and indexed. Here is what the numbers look like and what we are doing next month.

# **90-Day Metrics — Retention Conversation**

The 90-day review is the first retention conversation. The client has paid for three months. They need to see a clear connection between ClearSky and their business results. The rep’s job is to make that connection visible and concrete.

## **Internal checks**

- Full audit of intent rules — per-client overrides documented and confirmed

- Network model ingesting session data without anomalies

- All tiered session expiry windows functioning correctly

- ContentRadar publishing pipeline reviewed — content matching spiking keywords

## **Performance targets**

| **Metric** | **Target** | **Source** | **If below target** |
| --- | --- | --- | --- |
| Retargeting CTR — Emergency | 3–5% | Ad platform | Below 1.5% — escalate to Rory. Retention risk. |
| Retargeting CTR — Active Project | 2–4% | Ad platform | Below 1% — escalate to Rory. |
| Retargeting CTR — Comparison | 1.5–3% | Ad platform | Below 0.75% — review audience quality |
| Return visitor conversion lift | 20–30% above baseline | ClearSky analytics | Escalate to Rory if clean health layer — retention risk |
| One traceable closed job | Anon visit → bucket → retargeting → return → conversion → hub profile | Hub + ClearSky | Document what part of the trace is missing |
| Named hub profiles with history | 10–20 | Hub | Review hub event volume |
| Organic traffic lift on ContentRadar keywords | Measurable uplift vs month 1 baseline | ClearSky analytics | Review ContentRadar publishing cadence |
