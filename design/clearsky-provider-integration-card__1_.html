3B — Persona layer — inferPersona() runs on every pixel event

Every pixel event that passes Q1 and Q2 is also evaluated by inferPersona() non-blocking. The same event that updates the score also accumulates confidence across four persona dimensions. Persona never overrides bucket or score. It modifies the content manifest on top of them.

Four edge cases — inferPersona() must handle all four
1 double-count prevention — same event type not re-scored in same session
2 contradiction reset — delta ≥ 0.25 toward different class resets confidence to that delta only
3 hard cap enforcement — confidence never exceeds cap regardless of accumulation
4 atomic write — Prisma $transaction — partial updates never written to profile
Persona is the 7th Orchestrator input. Bucket, score, tier, session state, web history, ContentRadar state, and context (1–6) are evaluated first. Persona modifies the content manifest output — proof block, tone, CTA variant, urgency text, form length, content strip. A dimension is only applied when its confidence meets threshold. Below threshold it accumulates silently.
Q3 3C — Ad session start + content response events New events required
3C-i — ad_session_start · fires on every paid landing

When UTM source is a paid platform — Google, Facebook, YouTube, Bing — a dedicated session-open event fires before any behavioural events. Carries everything the ad decision put into the click. This is Stream A — it travels with the session and writes to the hub profile.

// Fires immediately on page load when paid UTM detected // Before scroll, dwell, or any behavioural event { "eventType": "ad_session_start", "utmSource": "google", // google | facebook | youtube | bing "utmCampaign": "spring-2026-emergency", "utmTerm": "emergency plumber Timmins", // keyword — Google only "utmContent": "headline-variant-b", // creative ID — if populated in campaign "matchType": "exact", // exact | phrase | broad — Google only "audienceSegment": "homeowner-35-54", // Meta only — if passed via utm_content "landingPage": "/emergency", "bucketPreload": "emergency", // keyword-driven bucket pre-load if keyword declared intent "device": "mobile", "sessionId": "sess_9d2f1a" }
Process dependency — not just a code dependency: utmContent carrying the creative variant requires the campaign manager to populate the tracking template in Google Ads or Meta before launch. ClearSky reads whatever arrives — but if the field is empty, Gap 2 stays open. This must be documented in the onboarding runbook as a required campaign setup step.
3C-ii — content response events · fires after dynamic content is served

Three events that close the personalization feedback loop. Without these, the system knows what caused the variant to be served — it does not know what the variant caused.

variant_served — fires when Orchestrator writes manifest to page
Fields: section (hero/proof/form/cta/urgency/content_strip) · variantBucket (which variant selected) · score_live at moment of serving · personaDimensionsApplied · adSessionId if paid source. Writes to hub + Cohort 2.
variant_impression — 3s in viewport with no interaction
Same fields as variant_served plus dwellBeforeBypass. Bypass confirmed — visitor saw the content and moved on. This is the most valuable signal: personalization decision actively not working. Routes to tuning candidates queue — does not affect score.
variant_engaged — existing click/form/tool events with variantBucket attached
Not a new event — variantBucket field added to all existing interaction events. hero_cta_click now carries which version of the hero was clicked. Minimal implementation cost — one field addition across existing events.
What these three events answer together
Stream A individual — did this specific session respond to the content decision made for them
Cohort 2 aggregate — which variant + persona + bucket combination converts across all sessions
Paid + organic split — does ad_session_start change content response vs organic arrival at same bucket
Persona validation — did the inferred persona lead to the right content decision or consistently produce bypasses
